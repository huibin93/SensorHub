1. 入口与配置 (大门与地基)
app/main.py
作用：项目的总入口;
看点：可以看是如何初始化 FastAPI 应用的，如何挂载路由 (include_router)，以及启动时的事件处理 (
on_startup
 中初始化数据库);
app/core/config.py
作用：配置文件;
看点：了解项目使用了哪些环境变量，数据库连接字符串 (
SQLITE_URL
) 是如何生成的，以及文件存储路径 (
STORAGE_DIR
) 在哪;
2. 数据模型 (骨架)
app/models/sensor_file.py
作用：核心数据表定义;
看点：
SensorFile
 类定义了文件的所有属性(状态、大小、设备类型等)，这是核心业务对象;
app/models/dictionary.py
作用：字典表定义;
看点：
DeviceModel
 (设备型号) 和 
TestType
 (测试类型) 的定义;
3. API 接口 (交互层)
app/api/v1/endpoints/files.py
作用：核心业务接口;
看点：这里是处理前端请求最密集的地方;重点看 
upload_file
 (上传)、
get_files
 (列表查询/筛选) 和 
get_file_data
 (读取解析后的数据);
app/schemas/api_models.py
作用：数据传输对象 (DTO);
看点：定义了前端请求发过来是什么样 (
Request
 模型)，后端返回去是什么样 (
Response
 模型);
4. 业务逻辑与服务 (核心处理)
app/crud/file.py
作用：数据库操作层;
看点：纯粹的数据库增删改查代码，将业务逻辑与 SQL 操作分离;
app/services/storage.py
作用：文件存储服务;
看点：如何保存上传的原始文件，以及如何生成文件路径;
app/services/parser.py
作用：文件解析服务;
看点：(未来核心) 这里将实现从原始二进制文件解析出数据的逻辑;
5. 测试与初始化 (工具)
app/core/database.py
：数据库连接和初始化逻辑;
test/test_db.py
：生成测试模拟数据的脚本，开发调试很有用;
